# Orchestra 0.0.17

**Released:** 2026-03-23

## What's new

### Heartbeat context window fix

The heartbeat had three compounding bugs that created a runaway feedback loop, consuming the entire context window:

1. **Cron job leaks** — cron jobs survive compaction at the platform level, but the agent's memory of them doesn't. Every compaction created a duplicate cron job (old one still firing + new one). Multiple heartbeats firing simultaneously filled the context window faster, causing more compaction, causing more duplicates.

2. **Recursive cron prompt** — the cron fired `/o heartbeat` which itself tried to schedule a new cron. Self-referential scheduling bomb.

3. **Post-compact hook demanded re-creation** — the hook unconditionally told the agent to run `/o heartbeat` after every compaction, creating yet another duplicate cron job.

**Fixes:**
- Cron prompt changed from `/o heartbeat` (recursive) to a minimal inline prompt that does at most 1 tool call and never invokes the skill
- Mandatory cron deduplication: `CronList` + `CronDelete` all existing jobs before creating new ones
- Post-compact hook no longer demands heartbeat re-creation (cron is already alive)

### Subagent delegation for heavy operations

Orchestra state updates (`/o checkpoint`, `/o docs`, `/o heartbeat`, post-work audits) now delegate to background subagents instead of doing file I/O inline. This protects the main context window from the overhead of reading/writing multiple `.orchestra/` files.

**Rule:** When an operation needs to read/write 3+ files, it spawns a background Agent. The main context only sees a one-line summary. The `.orchestra/` files themselves are the persistent log — no context is lost.

**Affected commands:**
- `/o heartbeat` — entire audit + scheduling runs in a subagent. Main context sees one line.
- `/o checkpoint` — all 6+ file writes happen in a subagent.
- `/o docs` — doc audit + cross-referencing + fixes happen in a subagent.
- Post-work audit — thread scanning happens in a subagent.

## Actions

Re-link repos to pick up the updated hooks (post-compact fix):

```
action: relink
prompt: "Repos need re-linking to get the heartbeat fixes. Run setup sync now?"
command: __ORCHESTRA_DIR__/setup sync "$_ORCH_ROOT"
```
