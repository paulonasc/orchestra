# Orchestra 0.0.20

**Released:** 2026-03-25

## What's new

### Agent-native interface rewrite

Agents ignored the old instruction-based rules (7 trigger-action pairs in CLAUDE.md tables). This release replaces them with mechanical enforcement and pattern-match triggers that work with how LLMs actually behave.

**PostToolUse nudge hook.** Counts code edits per session. After 10 edits without a checkpoint, the agent sees: "Orchestra: 12 edits since last checkpoint. Run /o checkpoint to save progress." Per-session counter files — concurrent agents have separate counters.

**Pattern-match triggers in skill description.** The `/o` skill description now includes: "Proactively suggest /o checkpoint when: the user says 'done', 'that's working', 'looks good', or 'all set'." Lives in the system prompt — permanent attention, never compacted.

**CLAUDE.md rules simplified.** From 30 lines (two tables of blocking/immediate rules) to 3 lines (pointer to /o and /o checkpoint). The heavy lifting moved to the nudge hook and skill description.

### Per-session write isolation

Checkpoints now write to per-session files only — no concurrent conflicts on shared state.

- `state/sessions/{id}.md` is the primary write target (all state goes here)
- `memory/YYYY-MM-DD.md` is append-only with `[session: {id}]` prefix
- `decisions/NNN-slug.md` uses unique filenames (no conflict)
- Shared files (`session-context.md`, `progress.yaml`, `MEMORY.md`) updated by merge-on-read at session start

### Structured recall template

`/o checkpoint` now prompts the agent through 6 categories before spawning the subagent: Code, Decisions, Research, Gotchas, Progress, Next. Prevents vague "built the footer" checkpoints.

### Repo-aware context filtering

Session-start hook filters by current repo basename:
- MEMORY.md entries tagged `[repo: api]` only show when working in the API repo
- Daily log entries filtered by repo name
- Untagged entries always show (global)

### Pre-compact safety nets

- Breadcrumb appended to daily log before compaction (trace even if agent never checkpoints)
- Session file stamped with compaction timestamp

### Eval system

Automated behavioral testing for hooks, skills, and agent compliance.
- Session runner using Claude Agent SDK (`query()`, not `claude -p` subprocess)
- LLM judge (OpenAI gpt-5.4-mini) for binary pass/fail behavioral checks
- 10 deterministic tests + 6 LLM behavioral evals
- See `evals/README.md` for full documentation

**Files changed:**
- `hooks/orchestra-post-tool-nudge.sh` — new PostToolUse hook
- `hooks/lib.sh` — new helpers: `get_session_id()`, `reset_edit_count()`
- `hooks/orchestra-session-start.sh` — session aggregation, repo-aware filtering, expanded session format
- `hooks/orchestra-pre-compact.sh` — breadcrumb + session snapshot
- `SKILL.md` — description rewrite, checkpoint target change, structured recall template
- `setup` — simplified `orchestra_rules_block()`, added PostToolUse to `install_hooks()`
- `evals/` — entire eval system (helpers, cases, hooks tests, README)

## Actions

```

prompt: "Major update — new nudge hook, simplified rules, per-session writes. Run setup sync now?"
command: __ORCHESTRA_DIR__/setup sync "$_ORCH_ROOT"
```
