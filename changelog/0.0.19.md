# Orchestra 0.0.19

**Released:** 2026-03-24

## What's new

### Concurrent session support

Orchestra now supports multiple agents working on the same repo at the same time. Previously, simultaneous sessions would stomp on each other's `state/session-context.md`. Now each session gets its own file at `state/sessions/{session-id}.md`.

**How it works:**
- `SessionStart` generates a unique session ID (PID + timestamp) and creates a per-session context file
- If other active sessions are detected, the agent gets a warning with details about concurrent sessions
- `Stop` hook cleans up the session file on exit (PID-matched so sessions don't accidentally remove each other's files)
- Stale sessions older than 24 hours are automatically pruned on session start
- `state/session-context.md` is still written as a copy for backwards compatibility (post-compact hook reads it)

**Files changed:**
- `hooks/lib.sh` — new functions: `generate_session_id()`, `list_active_sessions()`, `cleanup_stale_sessions()`
- `hooks/orchestra-session-start.sh` — generates session ID, creates session file, detects concurrent sessions
- `hooks/orchestra-stop.sh` — cleans up session file on exit
- `setup` — creates `state/sessions/` directory on init
- `SKILL.md` — updated session-context references for the session-scoped pattern

## Actions

Re-link repos to pick up the new hooks:

```
action: relink
prompt: "Repos need re-linking to get concurrent session support. Run setup sync now?"
command: __ORCHESTRA_DIR__/setup sync "$_ORCH_ROOT"
```
