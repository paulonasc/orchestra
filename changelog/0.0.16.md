# Orchestra 0.0.16

**Released:** 2026-03-23

## What's new

### Agent Awareness — 3-layer mid-session state tracking

Agents forget to update Orchestra state mid-session. This release adds three layers to fix it:

**Layer 1 — Instruction file injection (all agents)**
`setup link` now injects trigger-action rules into your repo's instruction file (CLAUDE.md, AGENTS.md, .cursor/rules, .opencode/instructions.md). Specific pairs like "after you commit → update session-context.md + daily log." Works across every agent. Zero runtime cost.

**Layer 2 — `/o heartbeat` with auto-schedule (Claude Code)**
Run `/o heartbeat` once — it schedules a lightweight inline cron (every 30 minutes) that checks state with near-zero context cost. The cron uses a minimal prompt — NOT a recursive `/o heartbeat` invocation — to prevent context window bloat. Always deduplicates existing cron jobs before creating new ones.

**Layer 3 — Channel server prototype (Claude Code, experimental)**
Event-driven heartbeat via Claude Code Channels. Fires on git commits instead of a timer. Blocked on Channels stabilizing (known bugs in v2.1.80-81). See `channels/README.md`.

### Improved briefing template

"When Done" checklist expanded: verification, docs update, and decision recording now included.

## Actions

After updating, re-link your repos to get the new instruction file rules:

```
action: relink
prompt: "Your repos need re-linking to get the new agent awareness rules. Run setup sync now?"
command: __ORCHESTRA_DIR__/setup sync "$_ORCH_ROOT"
```

For Claude Code users — enable automatic heartbeat:

```
action: suggest
prompt: "New: /o heartbeat auto-schedules state checks every 30 min. Try it now?"
command: /o heartbeat
```
