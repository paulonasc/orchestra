# Orchestra 0.0.18

**Released:** 2026-03-24

## What's new

### Blocking rules: research must be saved before coding

The agent awareness rules were rewritten. The old rules only had post-action triggers ("after you commit, update X") — but agents would finish research, build a rich mental model, then jump straight to implementation without saving anything to Orchestra.

**New structure:**
- **BLOCKING rules** — gates that must be completed *before* proceeding:
  - Before coding: write your plan/research to the thread first
  - Before spawning implementation subagents: update session-context
  - Before moving milestones: verify progress is current
- **IMMEDIATE rules** — the old post-action rules, plus: "after researching, save findings to conversation.md"
- Added reinforcement line: "If it's not in `.orchestra/`, it didn't happen."

Session-start hook also updated with the two new rules at the top of the rules block.

### Safe CLAUDE.md injection with start/end markers

Rules injection now uses `<!-- orchestra-rules-start -->` / `<!-- orchestra-rules-end -->` HTML comment markers instead of the old marker-to-EOF approach. This means:
- Content added **after** the Orchestra rules block is preserved on update
- `/o update` → `setup sync` safely replaces only the Orchestra block
- Legacy marker format (old installs) is auto-migrated on first sync

## Actions

Re-link repos to pick up the new rules:

```
action: relink
prompt: "Repos need re-linking to get the new awareness rules. Run setup sync now?"
command: __ORCHESTRA_DIR__/setup sync "$_ORCH_ROOT"
```
